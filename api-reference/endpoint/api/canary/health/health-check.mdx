---

Performs a health check on the application and external dependencies.

## Response

<ResponseField status="200" name="Healthy">
  <ResponseField name="status" type="string">
    Status of the system (e.g., "ok")
  </ResponseField>
  <ResponseField name="details" type="object">
    Details about individual dependency health
    <Expandable title="Details Object">
      <ResponseField name="nestjs-docs" type="object">
        Status of the NestJS documentation service
        <Expandable title="Service Status">
          <ResponseField name="status" type="string">
            Status of the service (e.g., "up", "down")
          </ResponseField>
        </Expandable>
      </ResponseField>
    </Expandable>
  </ResponseField>
</ResponseField>

## Example Response (Healthy)

```json
{
  "status": "ok",
  "details": {
    "nestjs-docs": {
      "status": "up"
    }
  }
}
```

## Example Response (Unhealthy)

```json
{
  "status": "error",
  "error": {
    "nestjs-docs": {
      "status": "down",
      "message": "Service unavailable"
    }
  }
}
```

## Error Responses

<ResponseField status="503" name="Service Unavailable">
  One or more health checks failed. The application or its dependencies are not healthy.
</ResponseField>

## Authentication

This endpoint does not require authentication and can be called by any client.
