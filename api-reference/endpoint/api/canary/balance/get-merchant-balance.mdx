Retrieves the merchant's own master balance (their earnings) available for payouts.

## Query Parameters

<ParamField query="merchantId" type="string">
  The ID of the merchant whose balance is being requested. If omitted, defaults to the authenticated merchant.
</ParamField>

## Response

<ResponseField status="200" name="Success">
  <ResponseField name="id" type="string">
    Unique identifier for the balance record
  </ResponseField>
  <ResponseField name="merchantId" type="string">
    The merchant's unique identifier
  </ResponseField>
  <ResponseField name="amount" type="number">
    Total balance amount
  </ResponseField>
  <ResponseField name="currency" type="string">
    The currency code
  </ResponseField>
  <ResponseField name="available" type="number">
    Amount available for immediate withdrawal
  </ResponseField>
  <ResponseField name="pending" type="number">
    Amount pending from recent transactions
  </ResponseField>
  <ResponseField name="createdAt" type="string">
    ISO 8601 timestamp when the balance record was created
  </ResponseField>
  <ResponseField name="updatedAt" type="string">
    ISO 8601 timestamp when the balance was last updated
  </ResponseField>
</ResponseField>

## Example Response

```json
{
  "id": "balance_merchant_456",
  "merchantId": "merchant_456",
  "amount": 50000,
  "currency": "ZAR",
  "available": 50000,
  "pending": 0,
  "createdAt": "2025-01-15T10:30:00.000Z",
  "updatedAt": "2025-01-15T10:30:00.000Z"
}
```

## Error Responses

<ResponseField status="401" name="Unauthorized">
  Invalid or missing authentication token
</ResponseField>

<ResponseField status="500" name="Internal Server Error">
  An unexpected error occurred on the server
</ResponseField>

## Authentication

This endpoint requires Bearer token authentication.
